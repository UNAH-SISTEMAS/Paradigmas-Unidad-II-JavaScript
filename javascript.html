La Web en 1993 — El Problema

A inicios de los 90, la World Wide Web era completamente estática: páginas HTML que solo mostraban texto e imágenes sin interacción alguna. Los formularios se enviaban al servidor para validarse, recargando toda la página ante cualquier error. Netscape Navigator, el navegador dominante, identificó la necesidad de un lenguaje que permitiera agregar comportamiento dinámico directamente en el navegador del usuario.

Los 10 Días que Cambiaron la Web

Netscape contrató a Brendan Eich en abril de 1995 con una misión: crear un lenguaje de scripting para el navegador. Eich escribió el primer prototipo en solo 10 días (del 6 al 15 de mayo de 1995). El lenguaje pasó por tres nombres antes de llegar al que conocemos hoy:

Mocha ──> LiveScript ──> JavaScript Mayo 1995 : Brendan Eich crea "Mocha" en 10 días Sep 1995 : Renombrado "LiveScript" al integrarse en Netscape Navigator 2.0 beta Dic 1995 : Renombrado "JavaScript" — alianza de marketing con Sun Microsystems (creadores de Java, muy popular en esa época) ⚠ JavaScript y Java son lenguajes completamente distintos. El nombre fue 100% marketing, no tiene relación técnica.
La Guerra de Navegadores y el Nacimiento de ECMAScript

Microsoft respondió creando JScript para Internet Explorer — una implementación propia de JavaScript con diferencias sutiles que generaron años de problemas de compatibilidad. Para resolverlo, Netscape envió JavaScript a ECMA International para su estandarización en 1996.

En 1997 nació el estándar oficial: ECMAScript (abreviado ES). JavaScript es la implementación más conocida de ECMAScript. El nombre "ECMAScript" evitó conflictos de marcas registradas con Sun/Oracle.

Node.js — JavaScript Conquista el Servidor (2009)

Ryan Dahl lanzó Node.js en 2009, llevando JavaScript al lado del servidor mediante el motor V8 de Google Chrome. Esto transformó JS de lenguaje exclusivamente de front-end a un lenguaje full-stack: un mismo desarrollador puede escribir el cliente y el servidor usando el mismo lenguaje y compartiendo código entre ambos.

¿Qué es una Variable?

Una variable es un contenedor con nombre que almacena un valor en la memoria del programa. En JavaScript las variables son de tipado dinámico: no se declara el tipo, y pueden contener distintos tipos de datos a lo largo del tiempo.

Hoisting — El Comportamiento "Mágico" de var

JavaScript eleva (hoist) las declaraciones al inicio de su scope antes de ejecutar. Con var la variable existe desde el inicio pero vale undefined. Con let y const existe una Zona Temporal Muerta (TDZ) donde accederla lanza un ReferenceError.

JavaScript — Hoisting 
// var — existe antes de declararse, pero vale undefined
console.log(saludo); // → undefined  (no lanza error)
var saludo = "Hola";
console.log(saludo); // → "Hola"

// let/const — Zona Temporal Muerta: lanza ReferenceError
// console.log(nombre); // ✗ ReferenceError: Cannot access before init
let nombre = "Ana";
console.log(nombre);  // → "Ana"

Operadores de Comparación
Regla fundamental en JavaScript
Siempre usa === (igualdad estricta) en lugar de == (igualdad débil). La igualdad débil realiza conversión de tipos automática y produce resultados sorpresivos que han causado incontables bugs.
JavaScript — Comparación 
// Igualdad DÉBIL (==) — convierte tipos antes de comparar
console.log(5 == "5");       // → true  ⚠ convierte "5" a número
console.log(0 == false);      // → true  ⚠
console.log("" == false);     // → true  ⚠
console.log(null == undefined);// → true  ⚠

// Igualdad ESTRICTA (===) — compara valor Y tipo
console.log(5 === 5);          // → true  ✓
console.log(5 === "5");        // → false ✓ (tipos distintos)
console.log(null === undefined);// → false ✓

// Desigualdad
console.log(5 != "5");         // → false (débil — iguales para ==)
console.log(5 !== "5");        // → true  (estricta — son distintos)

// Mayor, menor
console.log(10 > 5);            // → true
console.log(10 >= 10);          // → true
console.log(3 < 7);             // → true
console.log(3 <= 2);            // → false

JavaScript — Lógicos 
// && (AND) — verdadero si AMBOS son verdaderos

console.log(true  && false);   // → false

Operador Ternario

El operador ternario es una forma compacta de escribir un if/else simple en una sola línea. Su sintaxis es: condición ? valorSiTrue : valorSiFalse.

JavaScript — Operador ternario 
const edad = 17;

// Con if/else clásico
let acceso;
if (edad >= 18) {
  acceso = "Permitido";
} else {
  acceso = "Denegado";
}

// Con operador ternario — equivalente, más compacto
const acceso2 = edad >= 18 ? "Permitido" : "Denegado";
console.log(acceso2);  // → "Denegado"

// Otros usos comunes
const nota = 72;
const resultado = nota >= 60 ? "Aprobado" : "Reprobado";
const tipo      = nota % 2 === 0 ? "par" : "impar";

console.log(`Nota ${nota}: ${resultado} (${tipo})`);
// → "Nota 72: Aprobado (par)"

¿Qué es una Función?

Una función es un bloque de código con nombre, reutilizable, que realiza una tarea específica. En el paradigma funcional, las funciones son el elemento central del programa: en lugar de dar instrucciones paso a paso, se describen transformaciones a través de funciones que reciben datos y producen resultados sin modificar el estado externo.

Una buena función debe hacer una sola cosa y hacerla bien — es el principio de responsabilidad única aplicado a funciones.

Anatomía de una Función
JavaScript — Declaración de función 
//  palabra clave  nombre     parámetros
//       ↓           ↓            ↓        
function saludar(nombre) {
  return `Hola, ${nombre}!`;   // ← valor de retorno
}

// Invocar (llamar) la función
const mensaje = saludar("Ana");  // "Ana" es el argumento
console.log(mensaje);            // → "Hola, Ana!"
console.log(saludar("UNAH"));   // → "Hola, UNAH!"
Parámetro vs Argumento
El parámetro es la variable definida en la declaración (nombre). El argumento es el valor concreto que se pasa al invocar ("Ana"). Son la misma ranura, con dos nombres distintos según el momento.

En lugar de recibir un objeto completo y acceder a cada propiedad dentro de la función, puedes desestructurar directamente en la firma. Esto hace el contrato de la función más explícito.

function presentar({ nombre, carrera, promedio = "N/D" }) {

// → "Luis | Civil | Promedio: N/D"  (usa default)

// Destructuring de array en parámetros
function primeroYUltimo([primero, ...resto]) {

Scope Léxico — Las Funciones Recuerdan su Entorno

En JavaScript, el scope de una variable se determina por dónde está escrita en el código fuente, no por dónde se llama. Una función tiene acceso a todas las variables del scope donde fue definida, incluyendo los scopes externos que la rodean — esto forma la cadena de scope.

JavaScript — Cadena de scope 
const global = "soy global";

function exterior() {
  const enExterior = "soy exterior";

  function interior() {
    const enInterior = "soy interior";

    // interior VE todo: su scope + exterior + global
    console.log(enInterior);  // ✓ propio
    console.log(enExterior);  // ✓ scope padre
    console.log(global);      // ✓ scope global
  }

  interior();
  // console.log(enInterior);  // ✗ ReferenceError — exterior no ve hacia dentro
}

exterior();
¿Qué es un Closure?

Un closure ocurre cuando una función interna mantiene acceso a las variables de su scope externo incluso después de que la función externa ya terminó de ejecutarse. JavaScript "cierra sobre" el entorno de la función exterior, preservándolo en memoria.

JavaScript — Primer closure 
function crearSaludo(saludo) {
  // Devuelve una función — crearSaludo ya terminó,
  // pero la función interna recuerda "saludo"
  return function(nombre) {
    return `${saludo}, ${nombre}!`;
  };
}

const hola    = crearSaludo("Hola");
const buenos  = crearSaludo("Buenos días");

console.log(hola("Ana"));      // → "Hola, Ana!"
console.log(hola("Carlos"));  // → "Hola, Carlos!"
console.log(buenos("Toro"));  // → "Buenos días, Toro!"

// Cada closure tiene su propio "saludo" guardado
Estado Privado con Closures

Los closures permiten crear variables privadas: accesibles solo a través de las funciones expuestas, inaccesibles desde fuera. Esto es fundamental en el paradigma funcional para evitar estado mutable global.

JavaScript — Contador con estado privado 
function crearContador(inicio = 0) {
  let cuenta = inicio;  // ← variable privada — nadie accede desde fuera

  return {
    incrementar: () => ++cuenta,
    decrementar: () => --cuenta,
    reiniciar:   () => { cuenta = inicio; },
    valor:       () => cuenta,
  };
}

const c1 = crearContador();
const c2 = crearContador(100);

c1.incrementar(); c1.incrementar(); c1.incrementar();
console.log(c1.valor());  // → 3

c2.decrementar();
console.log(c2.valor());  // → 99

// Los contadores son independientes — cada closure tiene su propia "cuenta"
console.log(c1.valor());  // → 3  (no afectado por c2)
Fábrica de Funciones

Una fábrica de funciones (function factory) es una función que devuelve funciones especializadas. Es un patrón central del paradigma funcional: en lugar de repetir código, se generan versiones configuradas de una función base.

JavaScript — Fábricas de funciones 
// Fábrica de multiplicadores
const multiplicadorDe = factor => n => n * factor;

const doble    = multiplicadorDe(2);
const triple   = multiplicadorDe(3);
const decuplo  = multiplicadorDe(10);

console.log(doble(5));     // → 10
console.log(triple(5));    // → 15
console.log(decuplo(5));   // → 50

// Fábrica de validadores
const mayorQue = min => n => n > min;
const menorQue = max => n => n < max;

const esMayorDe18   = mayorQue(18);
const esMenorDe100  = menorQue(100);

console.log(esMayorDe18(20));   // → true
console.log(esMayorDe18(15));   // → false
console.log(esMenorDe100(50));  // → true

// Aplicación: filtrar con validadores generados
const edades = [12, 25, 17, 30, 15, 22];
const mayores = edades.filter(esMayorDe18);
console.log(mayores);            // → [25, 30, 22]

Estas tres funciones son los pilares del paradigma funcional en JavaScript. Todas reciben un callback (función) como argumento — son funciones de orden superior. Ninguna modifica el array original: siempre devuelven un nuevo resultado.

map — Transformar Cada Elemento

map crea un nuevo array aplicando una función a cada elemento del array original. La longitud del resultado siempre es igual a la del original.

// Sintaxis: array.map(elemento => transformación)

console.log(cuadrados);  // → [1, 4, 9, 16, 25]

const frutas     = ["manzana", "pera", "naranja"];
const mayusculas = frutas.map(f => f.toUpperCase());
console.log(mayusculas); // → ["MANZANA", "PERA", "NARANJA"]

// Transformar array de objetos
const estudiantes = [
  { nombre: "Ana",    nota: 85 },
  { nombre: "Carlos", nota: 72 },
  { nombre: "María",  nota: 90 },

reduce es la más poderosa de las tres. Recorre el array acumulando un resultado: el callback recibe el acumulador y el elemento actual. Con reduce puedes reimplementar map y filter, y resolver casi cualquier transformación.

JavaScript — reduce 
// Sintaxis: array.reduce((acumulador, elemento) => ..., valorInicial)

// Suma total
const total = notas.reduce((acc, n) => acc + n, 0);
console.log(total);                 // → 400

// Promedio
const promedio = notas.reduce((acc, n) => acc + n, 0) / notas.length;
console.log(promedio);              // → 80

// Máximo con reduce
const maximo = notas.reduce((max, n) => n > max ? n : max, notas[0]);
console.log(maximo);                // → 90

// Contar aprobados

JavaScript — Encadenamiento 
const estudiantes = [
  { nombre: "Ana",    nota: 85, activo: true  },
  { nombre: "Carlos", nota: 45, activo: true  },
  { nombre: "María",  nota: 90, activo: false },
  { nombre: "Pedro",  nota: 72, activo: true  },

  .filter(e => e.activo)             // solo activos: Ana, Carlos, Pedro, Laura
