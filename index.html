¿Qué es un Paradigma?

La palabra paradigma proviene del griego paradeigma, que significa "modelo" o "ejemplo". En ciencias, un paradigma es un conjunto de supuestos, conceptos, valores y prácticas que constituyen una forma de ver la realidad y de resolver problemas dentro de una disciplina.

En programación, un paradigma de programación es un estilo o enfoque fundamental para estructurar y pensar el código. Define cómo el programador concibe la solución: si el programa es una secuencia de instrucciones, un conjunto de objetos que interactúan, una serie de funciones matemáticas, o una colección de reglas lógicas.

Analogía
Piensa en un paradigma como una filosofía de diseño: del mismo modo que un arquitecto puede diseñar un edificio con filosofía minimalista, barroca o funcionalista, un programador puede resolver el mismo problema con paradigma imperativo, funcional u orientado a objetos.

Principales Paradigmas de Programación
1. Paradigma Imperativo

El programa describe cómo realizar la tarea: una secuencia explícita de instrucciones que cambian el estado del programa. Es el paradigma más cercano al funcionamiento físico de la computadora.

Python — Estilo imperativo 
# Calcular la suma de una lista de forma imperativa
numeros = [1, 2, 3, 4, 5]
suma = 0
for n in numeros:
    suma += n          # instrucción explícita que modifica estado
print(suma)            # → 15
2. Paradigma Procedimental

Extensión del imperativo que organiza el código en procedimientos o funciones reutilizables. El flujo del programa sigue una secuencia de llamadas a dichos procedimientos.

3. Paradigma Orientado a Objetos (POO)

Organiza el código en torno a objetos que combinan datos (atributos) y comportamiento (métodos). Los objetos interactúan entre sí mediante mensajes (llamadas a métodos).

Conceptos clave: Clases, Herencia, Encapsulamiento, Polimorfismo.

4. Paradigma Funcional

El programa se construye aplicando y componiendo funciones puras. Se evita el estado mutable y los efectos secundarios. Las funciones son tratadas como valores de primera clase (se pueden pasar como argumentos).

Python — Estilo funcional 
# La misma suma, con estilo funcional
numeros = [1, 2, 3, 4, 5]
resultado = sum(numeros)     # función pura, sin mutación de estado
print(resultado)              # → 15

# Usando reduce (funcional puro)
from functools import reduce
resultado = reduce(lambda a, b: a + b, numeros)
print(resultado)              # → 15
5. Paradigma Lógico

El programa describe qué debe ser verdadero, y el motor de inferencia determina cómo resolverlo. Representado principalmente por Prolog. No se especifica el flujo de control.

6. Paradigma Basado en Eventos

El flujo del programa es controlado por eventos externos: clics del usuario, respuestas de red, temporizadores. Es la base del desarrollo de interfaces gráficas y aplicaciones web interactivas.

Resumen — Mapa de Paradigmas
PARADIGMAS DE PROGRAMACIÓN ──────────────────────────────────────────── Imperativo ─┬─ Procedimental (C, Pascal) └─ Orientado a (Java, C++, Python, PHP) Objetos (POO) Declarativo ─┬─ Funcional (Haskell, Erlang, Python, JS) └─ Lógico (Prolog) Basado en eventos (JavaScript, PHP/web) Multiparadigma ── combina varios de los anteriores (Python, JavaScript, PHP, Ruby, Kotlin)

¿Qué es un Lenguaje Multiparadigma?

Un lenguaje multiparadigma es aquel que soporta de forma nativa más de un estilo o paradigma de programación, permitiendo al desarrollador elegir el enfoque más adecuado para cada problema, o incluso combinar varios enfoques dentro del mismo programa.

No significa que el lenguaje "mezcle" los paradigmas arbitrariamente, sino que ofrece las herramientas necesarias para programar bajo distintos estilos: puede tener clases (POO), funciones de primera clase y lambdas (funcional), estructuras de control clásicas (imperativo), y soporte para eventos.

Definición clave
Un lenguaje multiparadigma no impone un único estilo de pensamiento. Le da al programador la libertad — y la responsabilidad — de elegir el paradigma correcto para cada situación.
¿Por Qué Surgió el Multiparadigma?

Los paradigmas puros, aunque elegantes en teoría, presentan limitaciones prácticas:

  • La POO pura puede volverse rígida y burocrática para tareas simples de transformación de datos.
  • El funcional puro (como Haskell) tiene una curva de aprendizaje muy alta y puede ser poco intuitivo para modelar entidades del mundo real.
  • El imperativo puro escala mal en sistemas concurrentes y distribuidos modernos.

Los problemas reales rara vez encajan en una sola caja. Un sistema web necesita modelar entidades (POO), transformar datos eficientemente (funcional), reaccionar a eventos del usuario (eventos) y ejecutar lógica secuencial (imperativo). El multiparadigma nació de la necesidad pragmática.

Mismo Problema, Tres Enfoques

Filtrar los números pares de una lista y duplicarlos:

Python — Imperativo 
numeros = [1, 2, 3, 4, 5, 6]
resultado = []
for n in numeros:
    if n % 2 == 0:
        resultado.append(n * 2)
print(resultado)   # → [4, 8, 12]
Python — Funcional 
numeros = [1, 2, 3, 4, 5, 6]
resultado = list(map(lambda x: x * 2, filter(lambda x: x % 2 == 0, numeros)))
print(resultado)   # → [4, 8, 12]
Python — Comprensión de listas (híbrido) 
numeros = [1, 2, 3, 4, 5, 6]
resultado = [n * 2 for n in numeros if n % 2 == 0]
print(resultado)   # → [4, 8, 12]
Observación
Los tres fragmentos producen exactamente el mismo resultado. El multiparadigma no cambia qué hace el programa, sino cómo el programador lo expresa y lo estructura.

Tensiones del Multiparadigma

El multiparadigma también conlleva desafíos:

  • Consistencia de estilo: en un equipo, cada desarrollador puede preferir un paradigma distinto, generando código inconsistente.
  • Curva de aprendizaje: el estudiante debe dominar varios estilos, no solo uno.
  • Abuso de flexibilidad: mezclar paradigmas sin criterio puede producir código difícil de mantener.

Por eso, muchos equipos adoptan guías de estilo (como PEP 8 para Python) que establecen cuándo y cómo usar cada paradigma.

¿Qué es la CLI?

CLI (Command Line Interface) es una interfaz de texto donde el usuario interactúa con el sistema escribiendo comandos en una terminal o consola. No hay botones, imágenes ni ventanas: solo texto de entrada y texto de salida.

Los programas CLI son ideales para automatización, scripts de administración, procesamiento de datos, y como herramienta de aprendizaje porque permiten ver exactamente qué hace el código sin capas de abstracción visual.

FLUJO EN CLI ────────────────────────────────────── Usuario escribe comando en terminal ↓ Sistema operativo ejecuta el programa ↓ Programa lee entrada (stdin / argumentos) ↓ Programa procesa y genera resultado ↓ Resultado se muestra en consola (stdout) ↓ Programa termina (exit code 0 = éxito)
¿Qué es el Desarrollo Web?

El desarrollo web consiste en crear aplicaciones que se ejecutan sobre el protocolo HTTP/HTTPS, con un cliente (navegador) y un servidor. Existen dos grandes áreas:

  • Frontend: código que corre en el navegador del usuario. Controla la interfaz visual (HTML, CSS, JavaScript).
  • Backend: código que corre en el servidor. Gestiona la lógica de negocio, base de datos, autenticación (Python, PHP, JavaScript/Node.js).
FLUJO WEB (HTTP Request-Response) ────────────────────────────────────────────────── Navegador escribe URL y presiona Enter ↓ Navegador envía HTTP Request al servidor ↓ Servidor recibe la petición ↓ Código backend (Python/PHP/JS) procesa la solicitud ↓ Backend consulta base de datos (si es necesario) ↓ Backend genera respuesta HTML / JSON / XML ↓ Servidor envía HTTP Response al navegador ↓ Navegador renderiza la página o procesa el JSON

Clave de la Unidad
Un lenguaje multiparadigma moderno no solo mezcla estilos de programación: también puede ejecutarse en múltiples entornos. JavaScript, por ejemplo, usa exactamente el mismo lenguaje en el servidor (Node.js) y en el navegador — esto es una ventaja enorme en el desarrollo web moderno.

Python — Paradigma Orientado a Objetos
Python — POO 
class Animal:
    def __init__(self, nombre, sonido):
        self.nombre = nombre      # atributo de instancia
        self.sonido = sonido

    def hablar(self):             # método
        print(f"{self.nombre} dice: {self.sonido}")

class Perro(Animal):             # herencia
    def __init__(self, nombre):
        super().__init__(nombre, "Guau")

    def buscar(self, objeto):     # método propio
        print(f"{self.nombre} busca el {objeto}!")

rex = Perro("Rex")
rex.hablar()       # → Rex dice: Guau
rex.buscar("palo") # → Rex busca el palo!
Python — Paradigma Funcional
Python — Funciones de primera clase y lambdas 
# Las funciones son objetos — se pueden pasar como argumentos
def aplicar(func, lista):
    return [func(x) for x in lista]

cuadrado = lambda x: x ** 2
doble    = lambda x: x * 2

numeros = [1, 2, 3, 4, 5]
print(aplicar(cuadrado, numeros))  # → [1, 4, 9, 16, 25]
print(aplicar(doble,    numeros))  # → [2, 4, 6, 8, 10]

# map, filter, reduce — herramientas funcionales clásicas
pares = list(filter(lambda x: x % 2 == 0, numeros))
print(pares)   # → [2, 4]

Historia y Evolución

Creado por Brendan Eich en 1995 para Netscape Navigator en tan solo 10 días. Originalmente llamado Mocha, luego LiveScript, y finalmente JavaScript (nombre de marketing cercano a Java, aunque son lenguajes completamente distintos).

Estandarizado como ECMAScript (ES). Las versiones más importantes: ES5 (2009), ES6/ES2015 (clases, arrow functions, let/const, módulos), y actualizaciones anuales desde entonces. En 2009 nació Node.js, que lleva JavaScript al servidor.

JavaScript — Paradigma Orientado a Objetos (Clases ES6)
JavaScript — POO con clases 
class Vehiculo {
    constructor(marca, velocidadMax) {
        this.marca        = marca;
        this.velocidadMax = velocidadMax;
        this.velocidad    = 0;
    }

    acelerar(cantidad) {
        this.velocidad = Math.min(this.velocidad + cantidad, this.velocidadMax);
        return this;   // permite encadenar métodos
    }

    estado() {
        console.log(`${this.marca} viaja a ${this.velocidad} km/h`);
    }
}

class Auto extends Vehiculo {
    constructor(marca) {
        super(marca, 200);
    }
}

const miAuto = new Auto("Toyota");
miAuto.acelerar(80).acelerar(60).estado();  // Toyota viaja a 140 km/h

JavaScript — map, filter, reduce 
const productos = [
    { nombre: "Laptop",  precio: 15000 },
    { nombre: "Mouse",   precio: 350   },
    { nombre: "Monitor", precio: 6500  },
    { nombre: "Teclado", precio: 800   },

// Obtener nombres de productos que cuestan más de 1000

Creado por Rasmus Lerdorf en 1993-1994. Las siglas originales: Personal Home Page; hoy significa recursivamente PHP: Hypertext Preprocessor. Fue diseñado desde el principio para incrustarse dentro de páginas HTML y ejecutarse en un servidor web.

Actualmente impulsa aproximadamente el 77% de los sitios web con backend conocido, incluyendo WordPress, Drupal, Joomla, Facebook (comenzó en PHP) y Wikipedia.

PHP — Paradigma Orientado a Objetos (PHP 8+)
PHP — POO Moderna 
<?php
class Estudiante {
    public function __construct(
        private string $nombre,
        private float  $promedio
    ) {}  // PHP 8: constructor promotion

    public function estado(): string {
        return $this->promedio >= 60 ? "Aprobado" : "Reprobado";
    }

    public function __toString(): string {
        return "{$this->nombre} — {$this->estado()}";
    }
}

$estudiante = new Estudiante("María", 75.5);
echo $estudiante;  // → María — Aprobado
PHP — Paradigma Funcional
PHP — Funciones anónimas y array_map / array_filter 
<?php
$notas = [45, 78, 90, 55, 88, 30];

// Filtrar aprobados (>= 60) usando función anónima
$aprobados = array_filter($notas, fn($n) => $n >= 60);

// Aplicar bono de 5 puntos a cada aprobado
$conBono = array_map(fn($n) => min($n + 5, 100), $aprobados);

print_r($conBono);
// → [83, 95, 93]  (notas 78, 90, 88 con bono)

Beneficios del Multiparadigma
Flexibilidad · Expresividad · Industria · El futuro de los lenguajes
Objetivos
  • Articular con claridad los beneficios concretos del multiparadigma.
  • Relacionar el multiparadigma con la industria del software actual.
  • Comprender por qué aprender varios paradigmas es una ventaja competitiva.
Beneficio 1 — Flexibilidad de Expresión

El programador puede elegir el nivel de abstracción más adecuado para cada tarea. Tareas simples se escriben de forma simple (imperativa); sistemas complejos se modelan con objetos; transformaciones de datos se expresan funcionalmente.

Esto es como tener en la caja de herramientas tanto un martillo, como un destornillador, como una llave inglesa — y elegir la correcta para cada tornillo.

Beneficio 2 — Código más Legible y Mantenible
Python — Comparación de expresividad 
# Imperativo: describe CÓMO (verboso, más difícil de leer)
resultado = []
for nombre in nombres:
    if nombre.startswith('A'):
        resultado.append(nombre.upper())

# Funcional/declarativo: describe QUÉ (conciso, expresivo)
resultado = [n.upper() for n in nombres if n.startswith('A')]

# Ambos producen el mismo resultado — el segundo expresa la INTENCIÓN
Beneficio 3 — Adaptación a la Escala del Problema

Un script de 50 líneas no necesita clases ni jerarquías de herencia. Una aplicación empresarial de 500,000 líneas sí las necesita. El multiparadigma permite escalar el estilo junto con el problema:

Escala del proyectoParadigma típicamente dominante
Script de automatización (50-200 líneas)Procedimental / Funcional
API o servicio (500-5,000 líneas)Funcional + POO parcial
Aplicación empresarial (50,000+ líneas)POO + principios SOLID + funcional
Sistema reactivo / en tiempo realEventos + Reactivo + Funcional
Beneficio 4 — Relevancia en la Industria

Los tres lenguajes que estudiamos están entre los 10 más usados del mundo según el índice TIOBE y la encuesta anual de Stack Overflow. Dominan sectores enteros:

SECTORES INDUSTRIALES POR LENGUAJE (2024-2025) ──────────────────────────────────────────────── Python → IA / ML / Data Science / DevOps / Ciencia Empresas: Google, NASA, Netflix, Instagram JavaScript → Web frontend + fullstack + apps híbridas Empresas: Facebook, Twitter, Airbnb, LinkedIn PHP → Web tradicional + CMS + e-commerce Plataformas: WordPress (43% de la web), Magento, Drupal, Wikipedia
Beneficio 5 — Transferencia de Conocimiento entre Paradigmas

Aprender a pensar funcionalmente en Python te prepara para entender el estilo .map().filter().reduce() de JavaScript. Entender POO en PHP te prepara para Java, C# o Kotlin. Los paradigmas son portables — la habilidad trasciende el lenguaje.

El Futuro — Multiparadigma como Norma

Los lenguajes modernos más adoptados en los últimos años confirman la tendencia:

  • Kotlin (Android): POO + funcional + coroutines.
  • Swift (iOS/macOS): POO + funcional + value types.
  • Rust: funcional + imperativo + sistema de tipos estricto.
  • TypeScript: JavaScript con tipado + POO mejorada.

Los paradigmas puros son más relevantes en la academia para entender los fundamentos. En la industria, el multiparadigma pragmático es el estándar.

Historia de JavaScript

Brendan Eich creó JavaScript en 1995 mientras trabajaba en Netscape Communications. El lenguaje fue diseñado y escrito en tan solo 10 días con el objetivo de agregar interactividad a las páginas web estáticas de la época.

Su nombre original fue Mocha, luego LiveScript, y finalmente fue renombrado a JavaScript como estrategia de marketing para asociarlo con la popularidad de Java — aunque los dos lenguajes son completamente distintos en diseño y propósito.

AñoHito
1995Brendan Eich crea JavaScript en Netscape en 10 días.
1997Estandarización como ECMAScript 1 (ECMA-262).
2009Node.js: JavaScript llega al servidor (Ryan Dahl).
2015ES6 / ES2015: la mayor actualización del lenguaje — let, const, arrow functions, clases, módulos.
2016–hoyActualizaciones anuales (ES2016, ES2017…). JavaScript domina el desarrollo web.
Dato clave
JavaScript es el único lenguaje que corre nativo en todos los navegadores web. Esto lo hace indispensable: sin JavaScript, las páginas web serían documentos estáticos sin interacción.

Tipos de Datos

JavaScript es de tipado dinámico: no se declara el tipo al crear la variable; el intérprete lo determina según el valor asignado. Los tipos primitivos fundamentales son:

String (Cadena de texto)

Representa texto. Se escribe entre comillas simples, dobles, o template literals (backticks) que permiten incrustar expresiones con ${}.

JavaScript — String 
const saludo1 = "Hola, mundo";       // comillas dobles
const saludo2 = 'Hola, mundo';       // comillas simples
const nombre  = "Ricardo";
const saludo3 = `Hola, ${nombre}!`;  // template literal → "Hola, Ricardo!"

// Propiedades y métodos útiles
console.log(saludo1.length);        // → 11  (número de caracteres)
console.log(saludo1.toUpperCase()); // → "HOLA, MUNDO"
console.log(saludo1.includes("mundo")); // → true
console.log(saludo1[0]);            // → "H"  (acceso por índice)
Number (Número)

JavaScript usa un único tipo para todos los números: enteros y decimales. También existen valores especiales como NaN (Not a Number) e Infinity.

JavaScript — Number 
const entero   = 42;
const decimal  = 3.14;
const negativo = -7;

console.log(typeof entero);    // → "number"
console.log(typeof decimal);   // → "number"  (mismo tipo)

// Valores especiales
console.log(10 / 0);           // → Infinity
console.log("abc" * 2);        // → NaN  (Not a Number)
console.log(isNaN("abc" * 2)); // → true

// Métodos útiles
console.log(decimal.toFixed(1)); // → "3.1"  (1 decimal)
console.log(Math.round(4.7));    // → 5
console.log(Math.max(3, 8, 1));  // → 8
Boolean (Booleano)

Solo puede tener dos valores: true o false. Es el resultado de cualquier comparación lógica.

JavaScript — Boolean 
const esMayor  = true;
const esMenor  = false;

const esAdmin  = false;

console.log(typeof aprobado);  // → "boolean"
console.log(aprobado);         // → true

// Operadores lógicos
console.log(aprobado && esMayor);  // → true  (AND: ambos deben ser true)
console.log(aprobado || esAdmin);  // → true  (OR: al menos uno es true)

Null y Undefined

null representa la ausencia intencional de un valor — el programador lo asigna explícitamente para indicar "vacío".

undefined significa que una variable fue declarada pero no se le asignó ningún valor. JavaScript lo asigna automáticamente.

JavaScript — Null y Undefined 
// null: ausencia de valor intencional
let usuarioActivo = null;   // "aún no hay usuario seleccionado"
console.log(typeof usuarioActivo);  // → "object"  (particularidad histórica de JS)

// undefined: variable declarada sin valor
let resultado;
console.log(resultado);             // → undefined
console.log(typeof resultado);      // → "undefined"

// Comparación entre null y undefined
console.log(null == undefined);    // → true   (igualdad débil)
console.log(null === undefined);   // → false  (igualdad estricta: tipos distintos)
Array (Arreglo)

Un Array es una colección ordenada de valores. Los elementos se acceden por su índice (empezando en 0). Puede contener cualquier tipo de dato, incluso mezclados.

JavaScript — Array 
const frutas  = ["manzana", "banano", "naranja"];
const notas   = [85, 90, 78, 92];
const mezcla  = [1, "dos", true, null];  // tipos mixtos (válido)

// Acceso por índice (empieza en 0)
console.log(frutas[0]);        // → "manzana"
console.log(frutas[2]);        // → "naranja"

// Propiedad length
console.log(frutas.length);    // → 3

// Métodos comunes
frutas.push("uva");           // agrega al final → ["manzana","banano","naranja","uva"]
frutas.pop();                 // elimina el último → ["manzana","banano","naranja"]
console.log(frutas.includes("banano")); // → true
console.log(frutas.indexOf("naranja")); // → 2  (posición)

// Calcular promedio de notas
const suma     = notas.reduce((acc, n) => acc + n, 0);
const promedio = suma / notas.length;
console.log(`Promedio: ${promedio}`);  // → Promedio: 86.25

Operadores de Comparación

Siempre devuelven un Boolean. En JavaScript es fundamental distinguir entre == (igualdad débil, convierte tipos) y === (igualdad estricta, compara tipo y valor).

JavaScript — Operadores de Comparación 
let edad = 18;

console.log(edad >  17);   // → true   Mayor que
console.log(edad <  21);   // → true   Menor que
console.log(edad >= 18);   // → true   Mayor o igual
console.log(edad <= 17);   // → false  Menor o igual

// Igualdad débil (==) — convierte tipos antes de comparar
console.log(5  == "5");    // → true   (el string "5" se convierte a número)
console.log(0  == false);  // → true   (false se convierte a 0)

// Igualdad estricta (===) — compara tipo Y valor ← usar siempre
console.log(5  === "5");   // → false  (number vs string)
console.log(18 === 18);    // → true

// Desigualdad
console.log(edad != 21);   // → true   (débil)
console.log(edad !== 21);  // → true   (estricta — preferida)
Buena práctica
Usa siempre === y !== en lugar de == y !=. La igualdad débil (==) realiza conversiones de tipo implícitas que generan resultados inesperados y son una fuente común de bugs en JavaScript.
Condicionales

Los condicionales permiten ejecutar distintos bloques de código según si una condición es verdadera o falsa.

JavaScript — if / else if / else 
const nota = 75;
let calificacion;

if (nota >= 90) {
    calificacion = "Excelente";
} else if (nota >= 80) {
    calificacion = "Muy Bueno";
} else if (nota >= 70) {
    calificacion = "Bueno";
} else if (nota >= 60) {
    calificacion = "Suficiente";
} else {
    calificacion = "Reprobado";
}

console.log(`Nota: ${nota} → ${calificacion}`);
// → Nota: 75 → Bueno
JavaScript — Operador Ternario 
// Sintaxis: condición ? valor_si_true : valor_si_false
const edad    = 20;
const acceso  = edad >= 18 ? "Permitido" : "Denegado";
console.log(acceso);  // → "Permitido"

// Ideal para asignaciones simples de una sola condición
const precio    = 150;
const descuento = precio > 100 ? precio * 0.9 : precio;
console.log(`Precio final: $${descuento}`);  // → Precio final: $135
JavaScript — switch 
const dia = "lunes";

switch (dia) {
    case "lunes":
    case "martes":
    case "miércoles":
    case "jueves":
    case "viernes":
        console.log("Día de trabajo");
        break;
    case "sábado":
    case "domingo":
        console.log("Fin de semana");
        break;
    default:
        console.log("Día desconocido");
}
// → Día de trabajo
Bucles (Loops)

Los bucles permiten repetir un bloque de código mientras se cumpla una condición, o una cantidad determinada de veces.

for — cuando se conoce el número de repeticiones
JavaScript — for 
// for (inicialización; condición; incremento)
for (let i = 1; i <= 5; i++) {
    console.log(`Iteración ${i}`);
}
// → Iteración 1
// → Iteración 2  ...hasta 5

// Recorrer un array con for clásico
const colores = ["rojo", "verde", "azul"];
for (let i = 0; i < colores.length; i++) {
    console.log(colores[i]);
}
// → rojo, verde, azul
for...of — recorrer arrays de forma sencilla
JavaScript — for...of 
const estudiantes = ["Ana", "Luis", "María", "Carlos"];

for (const estudiante of estudiantes) {
    console.log(`Bienvenido/a, ${estudiante}`);
}
// → Bienvenido/a, Ana
// → Bienvenido/a, Luis  ...etc.

// Sumar todos los elementos
const numeros = [10, 20, 30, 40];
let total = 0;
for (const num of numeros) {
    total += num;
}
console.log(`Total: ${total}`);  // → Total: 100
while — cuando no se conoce el número de repeticiones
JavaScript — while y do...while 
// while: evalúa la condición ANTES de ejecutar
let intentos = 0;
while (intentos < 3) {
    console.log(`Intento #${intentos + 1}`);
    intentos++;
}
// → Intento #1, #2, #3

// do...while: ejecuta AL MENOS UNA VEZ antes de evaluar
let numero = 10;
do {
    console.log(`Número: ${numero}`);
    numero--;
} while (numero > 7);
// → Número: 10, Número: 9, Número: 8
