¿Qué es Software y qué es un Programa?

Software es el conjunto de programas, datos e instrucciones que permiten al hardware de una computadora realizar tareas específicas. Sin software, el hardware no puede hacer nada útil.

Un programa es una secuencia de instrucciones escritas en un lenguaje formal que la computadora puede interpretar y ejecutar para resolver un problema.

¿Qué es un Algoritmo?

Un algoritmo es un conjunto finito, ordenado y preciso de pasos que resuelven un problema o realizan una tarea.

CaracterísticaDescripción
FinitoTiene un inicio y un fin definidos.
PrecisoCada paso es claro y sin ambigüedad.
EfectivoProduce el resultado correcto para los datos dados.
GeneralFunciona para un conjunto amplio de casos, no solo uno.
Modelo: Entrada → Proceso → Salida
[ENTRADA] → [PROCESO] → [SALIDA] Ejemplo: Edad del usuario → Comparar edad ≥ 18 → "Es mayor de edad"
Programación Estructurada

Paradigma que organiza el código usando únicamente tres estructuras de control:

  1. Secuencia: Las instrucciones se ejecutan una tras otra, en orden.
  2. Decisión: Se elige entre dos o más caminos según una condición (if/else).
  3. Repetición: Un bloque se ejecuta varias veces (while, for).

Historia del Lenguaje C

Creado por Dennis Ritchie entre 1969 y 1973 en los Laboratorios Bell (Bell Labs). Diseñado originalmente para reescribir el sistema operativo UNIX. Evolucionó del lenguaje B (Ken Thompson), que vino del lenguaje BCPL.

Estandarizado en 1989 por ANSI como C89/C90, revisado posteriormente como C99 y C11.

El Proceso de Compilación
Código fuente (.c) → Compilador (gcc) → Ejecutable (.exe / a.out) Fases: 1. Preprocesador → procesa #include y #define 2. Compilación → traduce C a código ensamblador 3. Ensamblado → convierte ensamblador a código objeto (.o) 4. Enlazado → une el código objeto con bibliotecas → ejecutable
Tipos de Errores
1. Errores Sintácticos

El compilador los detecta; el programa no compila. Son los más fáciles de corregir.

C — Error sintáctico 
int main() {
    printf("Hola")   // ← falta el punto y coma ;
    return 0;
}
2. Errores Lógicos

El programa compila y ejecuta, pero produce resultados incorrectos. El compilador no los detecta.

C — Error lógico 
// Se suma en vez de calcular el promedio
int promedio = nota1 + nota2 + nota3;  // falta dividir entre 3
3. Errores en Tiempo de Ejecución

El programa compila correctamente pero falla durante la ejecución.

C — Runtime error 
int resultado = 10 / 0;  // división entre cero → crash

¿Qué es una Decisión en Programación?

Un programa toma decisiones cuando evalúa una condición y ejecuta diferentes instrucciones según el resultado (verdadero o falso).

En C, cualquier valor distinto de 0 es verdadero, y 0 es falso.

Estructura if
C — Sintaxis 
if (condicion) {
    // Se ejecuta SOLO si la condición es verdadera
}
[Condición] | ¿Es verdadera? / \ Sí No | | [Bloque if] [continúa] \ / [Siguiente instrucción]
Ejemplos
C — Número positivo, negativo o cero 
#include <stdio.h>

int main() {
    int numero;
    printf("Ingresa un número: ");
    scanf("%d", &numero);

    if (numero > 0) {
        printf("El número es positivo.\n");
    }
    if (numero < 0) {
        printf("El número es negativo.\n");
    }
    if (numero == 0) {
        printf("El número es cero.\n");
    }
    return 0;
}
Errores Comunes
C — Errores frecuentes 
// ✗ ERROR: asignación en lugar de comparación
if (edad = 18) { ... }    // ¡siempre verdadero!

// ✓ CORRECTO
if (edad == 18) { ... }

// ✗ ERROR: punto y coma termina el if vacío
if (edad >= 18);   // ← el if no hace nada
{
    printf("Mayor de edad\n");  // siempre se ejecuta
}

Sintaxis del switch
C — Estructura 
switch (expresion) {
    case valor1:
        // instrucciones
        break;
    case valor2:
        // instrucciones
        break;
    default:
        // si ningún caso coincide
}
ParteDescripción
switch(expresion)Evalúa la expresión (debe ser int o char).
case valor:Compara la expresión con ese valor exacto.
breakSale del switch al terminar el caso.
defaultSe ejecuta si ningún case coincide (opcional).
Fall-Through: El Error más Común
C — Fall-through indeseado 
int opcion = 1;
switch (opcion) {
    case 1:
        printf("Caso 1\n");
        // ¡FALTA break! → cae al caso 2
    case 2:
        printf("Caso 2\n");  // también se imprime
        break;
}
// Imprime: "Caso 1" y "Caso 2"
Ejemplos
C — Días de la semana 
#include <stdio.h>

int main() {

Ciclo while
Repetición controlada · Contadores · Acumuladores · Centinelas
Objetivos
  • Comprender el concepto de repetición controlada.
  • Usar la estructura while para resolver problemas iterativos.
  • Identificar y manejar contadores, acumuladores y centinelas.
Estructura while
C — Sintaxis 
while (condicion) {
    // Bloque que se repite mientras la condición sea verdadera
}
1. Evalúa la condición 2. Si es VERDADERA → ejecuta el bloque → regresa al paso 1 3. Si es FALSA → sale del ciclo y continúa el programa [Condición] → Falsa → [Continúa] | Verdadera ↓ [Bloque while] | (regresa a la condición)
Elementos Clave
Contador
C 
int i = 0;           // inicialización
while (i < 10) {    // condición
    printf("%d\n", i);
    i++;             // incremento — ¡nunca olvidarlo!
}
Acumulador
C — Suma de 1 a 100 
int i = 1, suma = 0;
while (i <= 100) {
    suma += i;   // acumula: suma = suma + i
    i++;
}
printf("Suma de 1 a 100: %d\n", suma);
Centinela
C — Terminar con -1 
int numero;
printf("Ingresa números (-1 para terminar):\n");
scanf("%d", &numero);

while (numero != -1) {     // -1 es el centinela
    printf("Ingresaste: %d\n", numero);
    scanf("%d", &numero);
}
printf("Ciclo terminado.\n");
Ciclo infinito
Si se olvida el incremento dentro del ciclo, la condición nunca se vuelve falsa y el programa se cuelga. Para detenerlo: Ctrl + C en la terminal.

Estructura do-while
C — Sintaxis 
do {
    // Bloque que se ejecuta AL MENOS UNA VEZ

Comparación: while vs do-while
Aspectowhiledo-while
Evaluación de condiciónAl inicioAl final
Ejecuciones mínimas0 (puede no ejecutarse)1 (siempre ejecuta al menos una vez)

Ejemplos
C — Menú repetitivo 
#include <stdio.h>

int main() {
    int opcion;

    do {
        printf("\n=== MENU ===\n");
        printf("1. Opción A\n");
        printf("2. Opción B\n");
        printf("3. Salir\n");
        printf("Selecciona: ");
        scanf("%d", &opcion);

        switch (opcion) {
            case 1: printf("Ejecutando Opción A...\n"); break;
            case 2: printf("Ejecutando Opción B...\n"); break;
            case 3: printf("Saliendo...\n");           break;
            default: printf("Opción inválida.\n");
        }
    } while (opcion != 3);

    return 0;
}
C — Validación de entrada 
#include <stdio.h>

int main() {
    int edad;

    do {
        printf("Ingresa tu edad (1-120): ");
        scanf("%d", &edad);

        if (edad < 1 || edad > 120) {
            printf("Edad no válida. Intenta de nuevo.\n");
        }
    } while (edad < 1 || edad > 120);

    printf("Edad registrada: %d\n", edad);
    return 0;
}
Transformar while en do-while
C — Equivalencia 
// Con while (necesita leer antes del ciclo)
printf("Ingresa un número positivo: ");
scanf("%d", &n);
while (n <= 0) {
    printf("Debe ser positivo. Intenta de nuevo: ");
    scanf("%d", &n);
}

// Con do-while (más natural)
do {
    printf("Ingresa un número positivo: ");
    scanf("%d", &n);
    if (n <= 0) printf("Debe ser positivo.\n");
} while (n <= 0);

Ejemplos
C — Tabla de multiplicar 
#include <stdio.h>

int main() {
    int numero;
    printf("¿Qué tabla quieres? ");
    scanf("%d", &numero);

    printf("\nTabla del %d:\n", numero);
    for (int i = 1; i <= 10; i++) {
        printf("%d x %d = %d\n", numero, i, numero * i);
    }
    return 0;
}
C — Factorial 
#include <stdio.h>

int main() {
    int       n;
    long long factorial = 1;

    printf("Ingresa un número: ");
    scanf("%d", &n);

    for (int i = 1; i <= n; i++) {
        factorial *= i;
    }

    printf("%d! = %lld\n", n, factorial);
    return 0;
}

Errores Comunes en Ciclos
C — Errores frecuentes 
// ✗ Off-by-one: imprime del 1 al 9, no del 1 al 10
for (int i = 1; i < 10; i++) { ... }   // debe ser i <= 10

// ✗ Ciclo infinito: i decrece, nunca llega a 10
for (int i = 0; i < 10; i--) { ... }

// ✗ Condición que nunca se cumple exactamente
for (int i = 0; i != 10; i += 3) { ... }
// i toma valores 0, 3, 6, 9, 12... nunca es exactamente 10
